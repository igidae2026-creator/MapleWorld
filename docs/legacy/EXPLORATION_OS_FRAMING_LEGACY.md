# Exploration OS Framing

> Legacy framing note.
> Keep as context only unless its rules are promoted into the top skeleton.

This file is a short working frame for ongoing MetaOS-related platform work.

## Position

Current runtime/platform work is valid.

It is not the top identity.

The higher identity is:

`MetaOS as an exploration OS`

## Interpretation Rule

When platform/runtime work and exploration-system properties compete, prefer the exploration-system properties.

Higher constraints:

- exploration-first design
- lineage visibility
- replayability
- append-only truth

Lower but still valid layer:

- runtime/platform convenience
- consumer onboarding speed
- adapter ergonomics
- operational reporting

## Practical Rule

Every new platform decision should be checked against these questions:

1. Does it preserve exploration history rather than overwrite it?
2. Does it preserve lineage and replayability?
3. Does it keep append-only truth as the source of authority?
4. Does it help exploration-scale autonomy rather than only runtime convenience?

If not, the platform layer is drifting above its proper rank and must be corrected.
